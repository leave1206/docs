**适用场景**：你的组织已经购置 ByteSpike，admin 已建好你的 lark 账号 + 充值过余额。你拿到这份文档，~5 分钟内能用上所有 admin 允许的 Claude / GPT / Gemini / DeepSeek / 豆包等模型。
</Note>

## 1. 装 DOSIA

<Tabs>
  <Tab title="内网员工">
    NAS 共享盘里取最新版：

    ```bash
    /Volumes/cloud/apps/mac/DOSIA-0.2.23-arm64.dmg   # Apple Silicon (M1/M2/M3/M4)
    /Volumes/cloud/apps/mac/DOSIA-0.2.23.dmg         # Intel Mac
    ```
  </Tab>
  <Tab title="公网下载">
    [github.com/leave1206/dosia-releases/releases/latest](https://github.com/leave1206/dosia-releases/releases/latest)
  </Tab>
</Tabs>

<Warning>
首次启动若报 **"文件已损坏"** 或 **"无法打开，因为来自身份不明的开发者"**，在 Terminal 跑：

```bash
xattr -cr /Applications/DOSIA.app
```

然后重新启动 DOSIA。这是 macOS Gatekeeper 对未公证 dmg 的默认拦截，不是 app 本身有问题。
</Warning>

## 2. 切到企业模式

<Steps>
  <Step title="打开 Settings">
    DOSIA 启动后顶部菜单：**DOSIA → Settings**（快捷键 `⌘,`）
  </Step>
  <Step title="切 Deploy Mode">
    左侧导航点 **Deploy Mode**，把模式从「个人版」切到 **「企业版」**
  </Step>
  <Step title="填企业服务地址">
    粘贴 admin 给你的地址 — 默认是：

    ```
    http://122.51.0.230:3012
    ```

    （admin 如果给了内网地址或自建域名，按 admin 的为准。）
  </Step>
  <Step title="点保存配置">
    页面下方蓝色「保存配置」按钮。DOSIA 会提示重启生效，跟着重启即可。
  </Step>
</Steps>

<Note>
该地址是 DOSIA **企业 web backend**（apps/web 部署，Tencent Cloud），**独立于** ByteSpike LLM Gateway (`https://llm.bytespike.ai`) — 两个服务不同机器、不同端口。Backend 负责 ByteSpike 配置同步、lark OAuth 代理、企业 workspace 管理；LLM Gateway 负责真实的模型调用。员工不需关心区别 — admin 给地址你填上即可。

如服务地址变更，admin 重新发新地址，员工 **Settings → Deploy Mode** 改字段 → 保存即可，**不用重装 DOSIA**。
</Note>

## 3. 企业 SSO 登录

<Steps>
  <Step title="确认 admin 已 grant">
    联系 admin 把你的 **lark 工号 / 邮箱**加入 ByteSpike 允许列表。Admin 在 [console.bytespike.ai](https://console.bytespike.ai) 或 lark 组管理后台操作 — 你不用动。
  </Step>
  <Step title="DOSIA 自动触发 lark OAuth">
    重启后 DOSIA 检测到企业模式，会自动弹 lark 授权页。第一次会跳出浏览器，授权完回 DOSIA。
  </Step>
  <Step title="登录成功">
    DOSIA 顶部右上角应显示你的 lark 头像 + 邮箱。
  </Step>
</Steps>

<Note>
没弹 lark 授权页 / 卡在「初始化」？跳到 [第 6 节常见问题](#6-常见问题) 检查。
</Note>

## 4. 自动获取允许的模型

登录后 DOSIA 会自动调 ByteSpike 拿你的可用模型列表 — 不用手动配 API key。

<Steps>
  <Step title="进 AI 模型 tab">
    **Settings → AI 模型**（左侧导航）。
  </Step>
  <Step title="确认看到「ByteSpike 共享」provider">
    AI 模型列表里应有一个名为 **「ByteSpike 共享」** 的 provider（admin 在企业模式下预配置的）。
  </Step>
  <Step title="确认模型列表">
    展开「ByteSpike 共享」，应看到 admin 给你 grant 的全部模型：

    - **Claude** 系列：Sonnet 4.6 / Opus 4.7 / Haiku 4.5
    - **OpenAI** 系列：GPT-5-4 / GPT-5-4-mini / GPT-5-2 等
    - **Gemini** 系列：Gemini 3.1 Pro / Gemini 3 Flash
    - **DeepSeek**：DeepSeek v4 Pro / v4 Flash / v3.2
    - **Volcengine / 豆包**：Doubao Seed 2.0 Pro 等
    - **图像 / 视频**：Nano Banana、Seedream 4、Sora 2、Veo 3.1 等（如 admin 已开通多模态权限）

    具体列表取决于你的组的权限配置 — 不一致请联系 admin。
  </Step>
</Steps>

## 5. 开始使用

<CardGroup cols={2}>
  <Card title="发文本消息" icon="message">
    主对话框顶部选模型 → 输入消息 → 回车发送。跨厂商模型用同一交互，ByteSpike 网关自动路由。
  </Card>
  <Card title="看图（vision）" icon="image">
    拖图进对话框（或粘贴），DOSIA 会自动调用 image-tools MCP 的 `analyze_image` 工具调用 vision 模型识图。
  </Card>
  <Card title="出图（image generation）" icon="palette">
    在对话框里要求生图（"画一只赛博朋克猫"），agent 会自动调 `generate_image` MCP 工具，走 Nano Banana / Seedream / GPT-Image 等。
  </Card>
  <Card title="出视频（video generation）" icon="video">
    类似图像 — 要求 "生成一段视频"，agent 调 `generate_video` MCP，走 Sora 2 / Veo 3.1 / Seedance 等。
  </Card>
</CardGroup>

<Note>
**计费透明**：每次模型调用的 USD 成本会显示在对话框下方的「credits」提示里。失败的调用（5xx / 网络错误）**不计费**。
</Note>

## 6. 常见问题

<AccordionGroup>
  <Accordion title='"ByteSpike 共享" provider 不出现'>
    可能原因：
    1. 企业服务地址填错 → 回 **Settings → Deploy Mode** 核对地址 + 重新保存
    2. lark grant 没生效 → 联系 admin 确认你的工号已加入 ByteSpike 允许列表
    3. SSO 授权过期 → DOSIA **设置 → 账户**点退出再重新登录
  </Accordion>

  <Accordion title='对话框报"余额不足"'>
    组织钱包余额耗尽。截图 + 联系 admin，admin 在 [console.bytespike.ai/billing](https://console.bytespike.ai/billing) 充值后立即恢复。
  </Accordion>

  <Accordion title='某模型报"无权限" / "model not allowed"'>
    Admin 没给你的组开通该模型权限。联系 admin 在 [llm.bytespike.ai/admin](https://llm.bytespike.ai/admin) → Users → 你的账户 → Allowed Models 勾上对应渠道组。
  </Accordion>

  <Accordion title='图像 / 视频生成 tab 看不到'>
    多模态权限是单独 grant 的（独立于文本模型）。Admin 在 console 给你的组加上 `image` / `video` capability 后，DOSIA 重启即可看到对应工具。
  </Accordion>

  <Accordion title='lark 授权后 DOSIA 卡死 / 反复登录'>
    清掉 DOSIA local state：

    ```bash
    rm -rf ~/Library/Application\ Support/DOSIA/auth
    ```

    然后重新启动 DOSIA。如果还不行，截图 + lark 私聊 admin。
  </Accordion>
</AccordionGroup>
