**厂商：** OpenAI
**Model ID：** `gpt-4o-mini`
**能力：** 128K 上下文 · 视觉 · 工具调用 · 流式
**计价：** 按 token，mini 档（[实时费率](https://bytespike.ai/pricing#text)）

GPT-4o mini 是传统 4o 系列里的小模型。在 prompt 长度可控、宁愿跑一百次便宜调用
也不想跑十次贵调用的高吞吐分类、抽取、路由场景里，它仍然是合适的选择。如果是
不需要视觉的新项目，5 系列的 mini 变体一般是更好的起点 ——
见 [GPT-5-mini](/zh/api-reference/text/gpt-5-mini) 和
[GPT-5.4-mini](/zh/api-reference/text/gpt-5-4-mini)。

## Request

```bash
curl https://llm.bytespike.ai/v1/chat/completions \
  -H "Authorization: Bearer $BYTESPIKE_API_KEY" \
  -H "content-type: application/json" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [
      {"role": "user", "content": "Classify the sentiment: Service was great."}
    ]
  }'
```

### Body 参数

| 字段 | 类型 | 必填 | 默认 | 说明 |
|---|---|---|---|---|
| `model` | string | 是 | — | `gpt-4o-mini` |
| `messages` | array | 是 | — | OpenAI chat 结构。 |
| `max_tokens` | integer | 否 | 模型上限 | 硬性上限。本模型最大值：16384。 |
| `temperature` | number | 否 | 1.0 | 范围 0.0–2.0。 |
| `top_p` | number | 否 | 1.0 | Nucleus 采样。 |
| `tools` | array | 否 | — | 支持 function calling。 |
| `tool_choice` | string \| object | 否 | `"auto"` | `"none"` / `"auto"` / `"required"` / `{"type":"function","function":{"name":"…"}}`。 |
| `response_format` | object | 否 | — | `{"type":"json_object"}` 启用 JSON 模式，`{"type":"json_schema",...}` 启用结构化输出。 |
| `stream` | boolean | 否 | false | SSE 流式。 |

## Response

```json
{
  "id": "chatcmpl-…",
  "object": "chat.completion",
  "model": "gpt-4o-mini",
  "choices": [

      "finish_reason": "stop"
    }
  ],
  "usage": {"prompt_tokens": 18, "completion_tokens": 1, "total_tokens": 19}
}
```

```python Python
from openai import OpenAI

client = OpenAI(
    base_url="https://llm.bytespike.ai/v1",
    api_key="$BYTESPIKE_API_KEY",
)

resp = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Classify the sentiment: Service was great."}],
)
print(resp.choices[0].message.content)
```

```javascript Node
import OpenAI from "openai"

const client = new OpenAI({
  baseURL: "https://llm.bytespike.ai/v1",

})

const resp = await client.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [{ role: "user", content: "Classify the sentiment: Service was great." }],
})
console.log(resp.choices[0].message.content)
```

## 流式

设置 `"stream": true` 以标准 OpenAI delta 格式走 SSE。首个 chunk 之前，估算
credits 会通过 HTTP 响应头返回。

## Prompt caching

OpenAI 采用 **自动 prompt caching** —— 没有 `cache_control` 参数。重复出现的
前缀（system prompt、工具定义）会透明命中缓存，并按
[价格表](https://bytespike.ai/pricing#text) 中标示的折扣价计费。在 agent
循环中保持 system prompt 与工具 schema 的稳定，以最大化缓存命中率。

## Errors

| Code | 触发条件 | 是否计费 |
|---|---|---|
| 400 | Body 校验失败 | 否 |
| 401 | key 缺失 / 已吊销 | 否 |
| 402 | 钱包余额不足 | 否 |
| 403 | Scope 被拒 / IP 不在白名单 | 否 |
| 422 | 参数不支持 | 否 |
| 429 | 触发限流 | 否 |
| 5xx | 上游问题 | 否（自动重试封装） |

## 何时选用

- 在传统代码上跑大批量分类、抽取或路由任务。
- 新项目优先考虑 [GPT-5-mini](/zh/api-reference/text/gpt-5-mini) 或 [GPT-5.4-mini](/zh/api-reference/text/gpt-5-4-mini)。
- 想要更高质量、可以接受更高成本的，见 [GPT-4o](/zh/api-reference/text/gpt-4o)。
- 想要最低档 5 系列里最快的，见 [GPT-5-nano](/zh/api-reference/text/gpt-5-nano)。
