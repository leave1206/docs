**厂商：** Anthropic
**Model ID：** `claude-opus-4-5`
**能力：** 1M context · tool use · vision · prompt caching · streaming · extended thinking
**价格：** 按 token，Opus 档（[实时价格](https://bytespike.ai/pricing#text)）

Opus 4.5 是把 1M token context 以 Opus 品质推到生产的那个模型。如果你
的 prompt 超过 200K 且需要对整体做推理，这就是 4 系列 Opus 家族的入口。
对新工作，建议选 [Opus 4.7](/zh/api-reference/text/claude-opus-4-7) —— 同样
的 1M context，输出更紧致、工具调用精度更高。4.5 保留是给已经针对它
验证过的团队继续使用。

## 请求

```bash
curl https://llm.bytespike.ai/v1/messages \
  -H "x-api-key: $BYTESPIKE_API_KEY" \
  -H "anthropic-version: 2023-06-01" \
  -H "content-type: application/json" \
  -d '{
    "model": "claude-opus-4-5",
    "max_tokens": 16384,
    "messages": [
      {"role": "user", "content": "Review this 800-page deposition for inconsistencies in the dates."}
    ]
  }'
```

### Body 参数

| 字段 | 类型 | 是否必填 | 默认 | 说明 |
|---|---|---|---|---|
| `model` | string | 是 | — | `claude-opus-4-5` |
| `messages` | array | 是 | — | 对话历史。最多 1M token 输入。 |
| `max_tokens` | integer | 是 | — | 硬上限。本模型最大：32768。 |
| `system` | string \| array | 否 | — | array 形式支持 `cache_control`。 |
| `temperature` | number | 否 | 1.0 | 范围 0.0–1.0。 |
| `top_p` | number | 否 | 1.0 | Nucleus sampling。 |
| `tools` | array | 否 | — | 支持。 |
| `tool_choice` | object | 否 | `{"type":"auto"}` | `auto` / `any` / `tool`（指定名）。 |
| `thinking` | object | 否 | — | Extended-thinking 预算。见 [Anthropic thinking 文档](https://docs.anthropic.com/claude/docs/extended-thinking)。 |
| `stream` | boolean | 否 | false | SSE 流式。 |

```javascript Node
import Anthropic from "@anthropic-ai/sdk"

const client = new Anthropic({
  baseURL: "https://llm.bytespike.ai/v1",
  apiKey: process.env.BYTESPIKE_API_KEY,
})

const message = await client.messages.create({
  model: "claude-opus-4-5",
  max_tokens: 16384,
  messages: [
    {
      role: "user",
      content: "Review this deposition for date inconsistencies.",
    },
  ],
})

const textBlock = message.content.find((b) => b.type === "text")
console.log(textBlock.text)
```

## Cache control

Cache control 是 4 系列中对成本最具杠杆效应的设置。1M token context
下，缓存命中可以让重复 agent 轮次的成本下降 10 倍。缓存价见
[pricing table](https://bytespike.ai/pricing#text)。

```json
{
  "model": "claude-opus-4-5",
  "system": [
    {
      "type": "text",
      "text": "<the 100K-token corpus you keep referring to>",
      "cache_control": {"type": "ephemeral"}
    }
  ],
  "messages": [...]
}
```

## 错误

| Code | 触发 | 是否计费 |
|---|---|---|
| 400 | Body 校验失败（`max_tokens` 过大等） | 否 |
| 401 | key 缺失 / 已撤销 | 否 |
| 402 | 钱包用尽（Opus 比 Sonnet 触发得更快） | 否 |
| 413 | 输入超过 1M token | 否 |
| 429 | 速率限制 | 否 |
| 5xx | 上游 provider 问题 | 否（自动重试信封） |

## 何时使用

- 200K–1M token 的长程推理（法律评审、代码库审计、多文档综合）。
- Sonnet 开始漏步骤的多步 prompt。
- 对新工作，建议选 [Opus 4.7](/zh/api-reference/text/claude-opus-4-7) —— 同样的 context window，输出更紧致。
- 中端成本 / 延迟，见 [Sonnet 4.6](/zh/api-reference/text/claude-sonnet-4-6)。
