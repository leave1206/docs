ByteSpike 用一把 ByteSpike key 提供 Gemini 家族 —— Gemini 2.5 Flash、
Gemini 2.5 Pro、Gemini 3.1 Pro —— 以及视频用的 Veo 3.1 / 3.1-Fast。ByteSpike 上的 Gemini 模型同时支持三种协议：

- **Gemini Native**（`/v1beta/models/{model}:generateContent`） —— 原样还原 Google 自家协议
- **OpenAI Chat Completions shim**（`/v1/chat/completions`） —— 给既有 openai-SDK 代码用
- **Anthropic Messages 翻译层**（`/v1/messages` 配上一个 Gemini 模型 ID） —— 给 Anthropic-SDK / Claude-Code 用户用；在 `protocols_aggregate` 中显示为 `anthropic_messages:translated`

选你客户端会的那种；同一个模型在三种协议下的行为是一致的。

## 端点覆盖

| 端点 | 路径 | 说明 |
|---|---|---|
| **Gemini Native** | `POST https://llm.bytespike.ai/v1beta/models/{model}:generateContent` | Google 原生协议。查询参数鉴权（`?key=...`）或 `Authorization: Bearer`。 |
| **Gemini Native（流式）** | `POST https://llm.bytespike.ai/v1beta/models/{model}:streamGenerateContent?alt=sse` | 返回 `data: {chunk}` 形式的 SSE 流。 |
| **OpenAI Chat Completions** | `POST https://llm.bytespike.ai/v1/chat/completions` | 翻译 shim。既有 openai-SDK 代码无需改动。 |
| **Anthropic Messages** | `POST https://llm.bytespike.ai/v1/messages`（带 Gemini 模型 ID） | 翻译 shim。Anthropic SDK / Claude Code 客户端通过这条路触达 Gemini。 |
| **视频生成** | `POST https://llm.bytespike.ai/v1/tasks/submit` | 走异步任务协议跑 Veo 3.1 / Fast。 |

## 鉴权

在 [Console](https://console.bytespike.ai) 创建一把 ByteSpike key 并绑定到
**`gemini-default`** 分组（或任何包含 Gemini 模型的分组）。key 用法：

- OpenAI / Anthropic shim 路径：`Authorization: Bearer $BYTESPIKE_API_KEY`
- `/v1beta` 原生路径：`?key=$BYTESPIKE_API_KEY` 查询参数，或 `Authorization: Bearer`

上游 Google 凭证（AI Studio API key、Vertex AI 服务账号，或绑定 Workspace 的
OAuth）配置在管理员侧，对你的客户端不可见。路由模型见 [channels](/zh/concepts/channels)。

## 快速开始

同一个 `gemini-2-5-pro` 调用，用三种协议各发一遍：

<CodeGroup>

```bash cURL (Gemini Native)
curl "https://llm.bytespike.ai/v1beta/models/gemini-2.5-pro:generateContent?key=$BYTESPIKE_API_KEY" \
  -H "content-type: application/json" \
  -d '{
    "contents": [{"parts": [{"text": "Hello, ByteSpike."}]}]
  }'
```

```python Python (openai SDK)
from openai import OpenAI

client = OpenAI(
    base_url="https://llm.bytespike.ai/v1",
    api_key=os.environ["BYTESPIKE_API_KEY"],
)

resp = client.chat.completions.create(
    model="gemini-2-5-pro",
    messages=[
        {"role": "user", "content": "Hello, ByteSpike."},
    ],
)

print(resp.choices[0].message.content)
```

```python Python (Anthropic SDK — 翻译层)
import anthropic

client = anthropic.Anthropic(
    base_url="https://llm.bytespike.ai",
    api_key=os.environ["BYTESPIKE_API_KEY"],
)

resp = client.messages.create(
    model="gemini-2-5-pro",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello, ByteSpike."}],
)

print(resp.content[0].text)
```

</CodeGroup>

## Gemini 专属能力

### Grounding（Google 搜索）

Gemini 的 grounding 工具 —— 用 Google 搜索做事实核查 —— 在 OpenAI shim 上
原样透传：`tools: [{"type": "google_search"}]`。Gemini Native 上写
`tools: [{"googleSearch": {}}]`。每次 grounding 调用按一段独立计费 + 上面的
每千次调用附加费。

### 长上下文

`gemini-2-5-pro` 接受最多 **2M 输入 token** —— ByteSpike 上最长的生产级
上下文。`gemini-3-1-pro` 是 1M。适合代码库级别审阅、长文 QA、多文档综述。

### Vision

三个 chat 模型都接受图像输入：Gemini Native 用 `inline_data`，OpenAI shim
用 `image_url` 块。`gemini-2-5-pro` 和 `gemini-3-1-pro` 原生支持 PDF。
图像输入计入 prompt token 预算。

### 函数调用

`/v1beta` 上是原生 `function_declarations`；OpenAI shim 上是标准 `tools`
数组（自动翻译）。三个模型都支持并行函数调用。

### 流式

OpenAI shim 上 `stream: true` 返回 OpenAI 格式的 SSE。Gemini Native 上
`:streamGenerateContent?alt=sse` 返回 Google 格式的 SSE。见
[流式](/zh/concepts/streaming)。

## 错误与速率限制

## 什么时候用 Gemini

- **gemini-2-5-flash** —— 便宜、快，通用 chat 上意外地能打。半价就接近 `gpt-4o-mini` 和 `claude-haiku-4-5`。
- **gemini-2-5-pro** —— 你需要 **2M 上下文窗口** 时。代码库级别任务、长文审阅、多文档综述 —— 200K 装不下的活。
- **gemini-3-1-pro** —— 最新旗舰。Gemini 家族内推理 + tool use 保真度最高。
- **veo-3-1 / veo-3-1-fast** —— 文生视频，动作真实；通过异步多模态任务流走。
