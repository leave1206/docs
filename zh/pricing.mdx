ByteSpike 每种 surface 只发布一套逐模型费率。无分层、无加价说明。
失败的请求不计费。

<Note>
**1 USD = 1,000,000 credits**（micro-USD 精度）。token 费率按每百万
token 报价。每次调用的费率按美元报价。
</Note>

## 费率从哪来

本文档每一行都来自生产网关的 admin 导出，每晚刷新。构建流水线读取
`channel_model_pricing` 的最新种子并生成本页；如果你需要程序化访问
同一份数据，直接打 `GET /api/pricing`（返回的 JSON 数组与下面表格行
一一对应）。

缓存定价约定：
- **Cache write**：大多数 provider 按 input 费率计费；Anthropic 按 1.25× input 计费。
- **Cache read**：所有 provider 一律 input 费率打 9 折（按 10% 计费）。
- **Web search 工具**（支持的 provider）：按每千次使用单独计费的附加项。

订阅 / 充值价格见 [bytespike.ai/pricing](https://bytespike.ai/pricing)。

## 工具端点

| 端点 | 成本 |
|---|---|
| `GET /balance` | 免费 |
| `POST /tasks/submit` | 成本 = 底层模型费率 × 时长 |

| `POST /tasks/cancel` | status=queued 时免费；running 时按比例计费 |

## 价格说明

- **失败不计费。** 任何非 2xx 响应都免费。唯一的例外：视频任务在
  `running` 之后被取消 —— 已消耗的 GPU 秒数会计费。
- **Cache write 费率**：大多数 provider 按 input 费率计 cache write。
  Anthropic 按 1.25× input 计 cache write。
- **Cache read 费率**：所有 provider 一律 9 折（按 10% 计费）。
- **Web search / grounding 工具**（支持的 provider）：按每千次使用
  单独计费，与 token 用量分开计。
- **图像 / 视频任务失败**：任务级别 100% 退款。
- **Chat 5xx**：不计费；网关在最外层做了自动重试。

## 程序化访问

```bash
curl https://llm.bytespike.ai/api/pricing \
  -H "x-api-key: $BYTESPIKE_API_KEY"
```

返回一个 JSON 数组，每款模型一项：

```json
{
  "model": "gpt-5-5",
  "category": "text",
  "vendor": "openai",
  "rates": {
    "input_per_1m": 5.00,
    "cache_write_per_1m": 5.00,
    "cache_read_per_1m": 0.50,
    "output_per_1m": 30.00,
    "currency": "USD"
  },
  "updated_at": "2026-05-08T04:30:00Z"
}
```

刷新频率为每天 UTC 04:30。客户端至少缓存 24 小时；`updated_at` 字段
告诉你底层网关数据最近一次刷新的时间。
