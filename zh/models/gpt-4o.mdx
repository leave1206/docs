`gpt-4o` 是 OpenAI 的多模态中档 —— vision、音频输入、tools、JSON mode。它跟 GPT-5 家族的唯一差异是**原生音频输入**，因此是无障碍、呼叫中心、语音流水线工作负载的默认选项。

## 快速开始

```bash
curl https://llm.bytespike.ai/v1/chat/completions \
  -H "Authorization: Bearer $BYTESPIKE_API_KEY" \
  -H "content-type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [{ "role": "user", "content": "Hello, ByteSpike." }]
  }'
```

## 什么时候用

- **音频输入流水线** —— 通话摘要、语音助手、无障碍。`gpt-4o` 是当前唯一直接吃音频的 GPT。
- **多模态中档** —— 需要 vision 又不需要推理时。
- **既有 `gpt-4o` pin 死的代码 drop-in** —— SDK 调用保持原样；ByteSpike 负责路由。

什么时候**不**用：

- 推理重的任务 —— 上 [gpt-5-4](/zh/models/gpt-5-4) 或更高。
- 高吞吐成本敏感 —— [gpt-4o-mini](/zh/models/gpt-4o-mini) 是它的十六分之一。
