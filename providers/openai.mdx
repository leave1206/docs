ByteSpike serves the OpenAI lineup — GPT-4o through GPT-5.5, plus
GPT-Image-2 for image generation and Sora 2 / 2-Pro for video —
behind a single ByteSpike key. Two chat protocols are wired up:
the ubiquitous **OpenAI Chat Completions** shape (`/v1/chat/completions`)
and the newer **Responses** shape (`/v1/responses`) that OpenAI's
Codex CLI and the Agents SDK now use.

## Endpoint coverage

| Endpoint | Path | Notes |
|---|---|---|
| **OpenAI Chat Completions** | `POST https://llm.bytespike.ai/v1/chat/completions` | Drop-in for any code already using the openai SDK. |
| **OpenAI Responses** | `POST https://llm.bytespike.ai/v1/responses` | The newer protocol; [Codex CLI](/codex-cli) and Agents SDK use it. Structured outputs + reasoning combine here. |
| **Image generation (sync)** | `POST https://llm.bytespike.ai/v1/images/generations` · `/v1/images/edits` | OpenAI-shape; serves GPT-Image-2 (and Nano-Banana via the same shape). |
| **Image / video generation (async)** | `POST https://llm.bytespike.ai/v1/tasks/submit` | Sora 2 / 2-Pro for video; long-running image batches. See [`/tasks/submit`](/api-reference/utility/tasks-submit). |

The same ByteSpike key works against every endpoint. Switch endpoints by path; the credential does not change.

<Note>
ByteSpike does **not** currently expose OpenAI's `/v1/embeddings`,
`/v1/audio/transcriptions`, `/v1/audio/speech`, `/v1/assistants`,
`/v1/files`, or `/v1/vector_stores` surfaces. If you depend on
those, call OpenAI direct for them; chat completions can still go
through ByteSpike.
</Note>

## Authentication

Issue a key from the [Console](https://console.bytespike.ai), then send it on every request as `Authorization: Bearer $BYTESPIKE_API_KEY`. Same scheme for every OpenAI-shape endpoint above.

The key must be bound to a routing group that includes the GPT
models. The console's [Test](https://console.bytespike.ai/dosia/models)
button confirms key + group + model viability before you wire it
into a client. See [Authentication](/authentication) for the full
per-key control list (quota, rate limits, IP allowlist).

## Quickstart

All four examples produce the same completion against `gpt-5-4` via Chat Completions.

<CodeGroup>

```bash cURL
curl https://llm.bytespike.ai/v1/chat/completions \
  -H "Authorization: Bearer $BYTESPIKE_API_KEY" \
  -H "content-type: application/json" \
  -d '{
    "model": "gpt-5-4",
    "messages": [
      { "role": "user", "content": "Hello, ByteSpike." }
    ]
  }'
```

```python Python (openai SDK)
from openai import OpenAI

client = OpenAI(
    base_url="https://llm.bytespike.ai/v1",
    api_key=os.environ["BYTESPIKE_API_KEY"],
)

resp = client.chat.completions.create(
    model="gpt-5-4",
    messages=[
        {"role": "user", "content": "Hello, ByteSpike."},
    ],
)

print(resp.choices[0].message.content)
```

```javascript Node (openai SDK)
import OpenAI from "openai"

const client = new OpenAI({
  baseURL: "https://llm.bytespike.ai/v1",
  apiKey: process.env.BYTESPIKE_API_KEY,
})

const resp = await client.chat.completions.create({
  model: "gpt-5-4",
  messages: [
    { role: "user", content: "Hello, ByteSpike." },
  ],
})

console.log(resp.choices[0].message.content)
```

```python Responses API (Codex)
from openai import OpenAI

client = OpenAI(
    base_url="https://llm.bytespike.ai/v1",
    api_key=os.environ["BYTESPIKE_API_KEY"],
)

resp = client.responses.create(
    model="gpt-5-4",
    input="Hello, ByteSpike.",
    reasoning={"effort": "medium"},
)

print(resp.output_text)
```

</CodeGroup>

## OpenAI-specific features

### Reasoning

GPT-5 family exposes a `reasoning_effort` parameter (Chat Completions) or `reasoning.effort` (Responses) controlling how many internal tokens the model spends reasoning before producing the answer. Values: `low` / `medium` / `high`.

Reasoning tokens count toward output billing. See [credits and billing](/concepts/credits-and-billing).

### Structured outputs

`response_format: { type: "json_schema", json_schema: {...} }` returns JSON that conforms to your schema, with provider-side validation. Supported on GPT-5-2 and newer. Passed through verbatim — ByteSpike doesn't modify the schema.

### Web search

GPT-5-2 / 5-4 / 5-4-pro / 5-5 support OpenAI's web search tool. Add `tools: [{ "type": "web_search" }]` to your request. Each tool invocation bills as a separate output segment + the per-1k-uses web-search surcharge.

### Function calling

Standard `tools` array plus `tool_choice` work on every chat model. Parallel tool use is supported on the GPT-5 family. Passed through unchanged.

### Streaming

`stream: true` on either chat protocol returns SSE. Mid-stream errors close the connection with an `error` event in the final data frame and bill **zero** credits, even if partial tokens were already streamed. See [Streaming](/concepts/streaming).

## Errors and rate limits

Identical contract to other ByteSpike protocols — see [Errors reference](/concepts/errors-reference) for the full status matrix and [Rate limits](/concepts/rate-limits) for the per-key budget model.

| Status | Trigger | Billed? |
|---|---|---|
| 400 / 401 / 402 / 403 / 429 | Bad input / auth / quota | No |
| 500 / 502 / 503 / 504 | Upstream | No — gateway retries on next channel in the same group before surfacing |

Per-key rate limits live on every API key (see [Authentication](/authentication)); per-account concurrency lives on the subscription tier (see [pricing](/pricing)).

## When to use which GPT

- **gpt-4o-mini** — classification, routing, structured extraction at scale. Cheap and fast.
- **gpt-4o** — multimodal workhorse for vision + tool use without reasoning.
- **gpt-5-nano** / **gpt-5-mini** — when you need reasoning but want to keep latency / cost down.
- **gpt-5-2** — the default mid-tier for fresh-fact tasks; first GPT-5 with web search.
- **gpt-5-3-codex** — code-tuned; pair with [Codex CLI](/codex-cli).
- **gpt-5-4** — production workhorse. Vision, tools, structured outputs, reasoning, web search — together at a Sonnet-like price.
- **gpt-5-4-pro** — for hard reasoning that warrants the extra compute.
- **gpt-5-5** — the 1M-context flagship. Codebase-scale refactors, long-document review.
