ByteSpike routes through seven upstream platforms: Anthropic, OpenAI, Google (Gemini), DeepSeek, plus the Volcengine CodePlan subscription bundle which itself wraps Doubao, GLM, Kimi, and MiniMax. Each platform has different strengths, different protocols, different billing models. This page is the side-by-side reference for picking between them.

For *why* the same model can show up on multiple platforms with different prices, read [channels](/concepts/channels). For the underlying protocol contracts, read [endpoint types](/concepts/endpoints).

## Billing model comparison

| Platform | How you pay | Predictability | Best fit |
|---|---|---|---|
| Anthropic | Per-token USD (rates published per-model) | Variable | When usage is bursty or hard to forecast |
| OpenAI | Per-token USD | Variable | Same |
| Gemini | Per-token USD (AI Studio); per-token + Cloud project quota (Vertex) | Variable | Enterprise: Vertex Project quota pins the upper bound |
| DeepSeek | Per-token USD; among the cheapest tier on ByteSpike | Variable | High-volume reasoning at a low dollar-per-quality |
| Volcengine CodePlan | **Subscription** ¥299/month upstream; admin amortizes across team | **Flat** | Teams running mixed Chinese-LLM workloads where finance wants a fixed number |

The break-even for CodePlan vs paying each Chinese vendor separately: ~100K mixed-vendor requests / month. Below that, per-token providers win; above, CodePlan wins. See [/providers/volcengine-codeplan](/providers/volcengine-codeplan) for the math.

## When to pick which platform

A short decision aid for the most common scenarios. None of these are exclusive — you can route different requests to different platforms from the same key.

| Scenario | Pick |
|---|---|
| Building an agent that needs `tool_use` + `thinking` + `cache_control` | **Anthropic** (Sonnet 4.6 or Opus 4.7) |
| Codex CLI integration | **OpenAI** (gpt-5-3-codex via Responses) |
| Multimodal with native audio input | **OpenAI** (gpt-4o is the only audio-in model on ByteSpike) |
| 1M+ context, lowest cost | **Gemini** (2.5 Flash for 1M, 2.5 Pro for 2M) |
| Web search + reasoning at premium tier | **OpenAI** (gpt-5-4-pro) or **Anthropic** (Opus 4.6 + 4.7) |
| Reasoning chain on a budget | **DeepSeek** (V4 Pro; ~10× cheaper than gpt-5-4 at comparable quality) |
| Chinese-LLM team with mixed vendor usage | **Volcengine CodePlan** subscription |
| OpenAI-SDK code that needs to swap to a cheaper Chinese model | **DeepSeek** Chat Completions or **CodePlan** Chat Completions |
| Anthropic-SDK agent that needs a Chinese-LLM fallback | **DeepSeek** Anthropic Messages (only Chinese vendor with anthropic-compat HTTP API today) |

| `/v1/tasks/submit` (async multimodal) | — | ✅ Sora | ✅ Veo | — | ✅ Seedance |

The same ByteSpike key authorizes every protocol surface. Switching protocols means switching the path, not the credential.

## Channel-vs-channel for one platform

For Anthropic and Gemini specifically, ByteSpike maintains multiple **channels per platform**:

- **Anthropic**: Anthropic API direct, AWS Bedrock (multiple regions)
- **Gemini**: Google AI Studio upstream is the primary channel; admin-configured Vertex / Workspace OAuth channels can sit alongside in the same group when wired up

These channels speak the same protocol but differ on pricing, region, audit-trail integration, and quota structure. See [channels](/concepts/channels) for the routing model and [platforms vs channels](/concepts/platforms-vs-channels) for the term disambiguation.

## What this comparison does NOT cover

- **Latency tiers** — production p50 / p90 / p99 latency profiles vary per channel per region. The Console latency dashboard is the source of truth.
- **Rate-limit defaults** — every platform has its own RPM / TPM ceilings on free / paid tiers; ByteSpike's own per-plan limits sit on top. See the [pricing footer](https://bytespike.ai/pricing).
- **Per-model quirks** — fine-grained capability differences between Sonnet 4.5 vs 4.6, or GPT-5.4 vs 5.4-pro, live on the per-model pages under [/models](/models).
